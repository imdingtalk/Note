mongodb副本集搭建

June 23, 2019

介绍

副本集是mongoDB副本所组成的一个集群，集群中没有特定的主库，主库是选举产生，如果主库down了，会再选举出一台主库

正常状态下拓扑故障切换过程

副本集有以下特点：1. 最小构成是：primary，secondary，arbiter，一般部署是：primary，2 secondary。2. 成员数应该为奇数，如果为偶数的情况下添加arbiter，arbiter不保存数据，只投票。3. 最大50 members，但是只能有 7 voting members，其他是non-voting members。

部署

下载mongod：https://www.mongodb.com/download-center/community加入到系统路径：chmod +x mongod && cp mongod /usr/bin/创建用户并设置密码： useradd mongod && passwd mongod编辑配置文件  

# mongod.conf

# for documentation of all options, see:
#   http://docs.mongodb.org/manual/reference/configuration-options/
# where to write logging data.
systemLog:
  destination: file
  logAppend: true
  path: /var/log/mongodb/mongod1.log
# Where and how to store data.
storage:
  dbPath: /var/lib/mongo1
  journal:
    enabled: true
#  engine:
#  mmapv1:
#  wiredTiger:
# how the process runs
processManagement:
  fork: true  # fork and run in background
  pidFilePath: /var/run/mongodb/mongod1.pid  # location of pidfile
  timeZoneInfo: /usr/share/zoneinfo
# network interfaces
# bindIp默认是127的，需要改成本机地址或者0.0.0.0，以远程连接，端口如果是单机部署，需要区分不同的端口
net:
  port: 27017
  bindIp: 127.0.0.1,172.16.13.44  # Enter 0.0.0.0,:: to bind to all IPv4 and IPv6 addresses or, alternatively, use the net.bindIpAll setting.
#security:
#  authorization: enabled
#  keyFile: /tmp/key.file
#operationProfiling:
#打开副本集功能
replication:
  replSetName: "rs0"
#sharding:
## Enterprise-Only Options
#auditLog:
#snmp:

将mongo添加到系统服务，以便管理   ，vim /usr/lib/systemd/system/mongod1.service


#单机部署的实例如下
[Unit]
Description=MongoDB Database Server
After=network.target
Documentation=https://docs.mongodb.org/manual

[Service]
User=mongod
Group=mongod
Environment="OPTIONS=-f /etc/mongod1.conf"
EnvironmentFile=-/etc/sysconfig/mongod
ExecStart=/usr/bin/mongod $OPTIONS
#多机部署的话，注意修改这里的配置
ExecStartPre=/usr/bin/mkdir -p /var/run/mongodb1
ExecStartPre=/usr/bin/chown mongod:mongod /var/run/mongodb1
ExecStartPre=/usr/bin/chmod 0755 /var/run/mongodb1
PermissionsStartOnly=true
PIDFile=/var/run/mongodb/mongod1.pid
Type=forking
# file size
LimitFSIZE=infinity
# cpu time
LimitCPU=infinity
# virtual memory size
LimitAS=infinity
# open files
LimitNOFILE=64000
# processes/threads
LimitNPROC=64000
# locked memory
LimitMEMLOCK=infinity
# total threads (user+kernel)
TasksMax=infinity
TasksAccounting=false
# Recommended limits for for mongod as specified in
# http://docs.mongodb.org/manual/reference/ulimit/#recommended-settings
[Install]
WantedBy=multi-user.target

启动服务， systemctl  start mongod。然后连接服务器，mongo --port 27019

use admin
rs.initiate()
#添加副本集成员
rs.add('172.16.13.xx:27018')
#删除节点
rs.remove('172.16.13.xx:27018')
#故障切换，主动关闭主库
use admin
db.shutdownServer()

安全认证

在生产环境，建议配置至少一个安全认证方式，我这里采用keyfile的认证方式在配置文件添加安全选项配置  

#创建一个秘钥
openssl genrsa -out ~/key.file 1024
#mongo只识别密钥，删除生成文件的首行和尾行
#修改权限
chmod  600 ~/key.file&&chown  mongod.mongod /tmp/key.file
#新增配置
security:
   keyFile: xxx.key
   clusterAuthMode: keyFile
   authorization: enabled #由于开启验证必须添加用户
#创建用户

use admin
db.createUser( {user: "cluster",pwd: "cluster",roles: [ { role: "clusterAdmin", db: "admin" } ]})
db.createUser( {user: "admin",pwd: "123456",roles: [ { role: "userAdminAnyDatabase", db: "admin" } ]})
#查看用户的角色和拥有的权限
db.getUser('cluster')
db.getRole("clusterAdmin",{ showPrivileges: true } )
#授予新的角色
use admin 
db.grantRolesToUser(
    "cluster",
    [
      { role: "clusterAdmin", db: "admin" }
    ]
)
#撤销角色
db.revokeRolesFromUser(
    "cluster",
    [
      { role: "userAdminAnyDatabase", db: "admin" }
    ]
)
