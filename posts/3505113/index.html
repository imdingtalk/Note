基于snmp的网络设备的监控.md

January 4, 2020

介绍

公司当前环境下有基于prometheus的监控体系，覆盖了所有的服务器和虚拟机，但是还没有包含网络设备，例如交换机或者防火墙，因此，需要补充网络设备监控部分

相关信息

prometheus： 监控系统，负责指标的收取和一些告警规则的定义snmp-exporter: 用于通过snmp协议暴露交换机的相关指标SNMP Exporter Config Generator： 此配置生成器使用NetSNMP解析MIB，并使用它们为snmp_exporter生成配置–帮助生成snmp的配置文件MIB和OID： MIB是管理信息库的缩写，它是用于管理通信网络中的实体的数据库。数据库是分层的（树形结构），并且每个条目都通过对象标识符（OID）进行寻址snmp协议： SNMP 是专门设计用于在 IP 网络管理网络节点（服务器、工作站、路由器、交换机及HUBS等）的一种标准协议

snmp-exporter generator

Prometheus team特此准备了snmp-exporter generator一个专门用于生成snmp-exporter的工具Generator: Generator

安装
 sudo yum install gcc gcc-g++ make net-snmp net-snmp-utils net-snmp-libs net-snmp-devel # RHEL-based distros
go get github.com/prometheus/snmp_exporter/generator
cd ${GOPATH-$HOME/go}/src/github.com/prometheus/snmp_exporter/generator
go build
make mibs
使用
export MIBDIRS=mibs
./generator generate
它会根据generator.yml的配置来生成配置文件，generator.yml的配置就简单多了

generator.yml
modules:
  huawei:
    walk:
      - hwDev
      - 1.3.6.1.2.1.1.3   #sysUpTime
      - 1.3.6.1.2.1.31.1.1.1.10 #ifHCOutOctets
      - 1.3.6.1.2.1.2.2.1.14 #ifInErrors
      - 1.3.6.1.2.1.2.2.1.20 #ifOutErrors
      - 1.3.6.1.2.1.31.1.1.1.6 #ifHCInOctets
      - 1.3.6.1.2.1.2.2.1.21 #ifOutQLen
      - 1.3.6.1.2.1.2.2.1.13 #ifInDiscards
      - 1.3.6.1.2.1.2.2.1.19 #ifOutDiscards
      - 1.3.6.1.2.1.2.2.1.7 #ifAdminStatus
      - 1.3.6.1.2.1.2.2.1.8 #ifOperStatus
      - 1.3.6.1.2.1.2.2.1.2 #ifDescr
      - 1.3.6.1.2.1.31.1.1.1.1 #ifName
      - 1.3.6.1.2.1.2.2.1.5 #ifSpeed
      - 1.3.6.1.2.1.31.1.1.1.15 #ifHighSpeed
      - 1.3.6.1.2.1.31.1.1.1.18 #ifAlias
      - 1.3.6.1.2.1.2.2.1.3 #ifType
      - 1.3.6.1.2.1.31.1.1.1.11 #ifHCOutUcastPkts
      - 1.3.6.1.2.1.31.1.1.1.7 #ifHCInUcastPkts
    ##定义数据中需要的额外的labels
    lookups:
      - source_indexes: [ifIndex]
        lookup: ifAlias
      - source_indexes: [ifIndex]
        lookup: ifDescr
      - source_indexes: [ifIndex]
        # Use OID to avoid conflict with Netscaler NS-ROOT-MIB.
        lookup: 1.3.6.1.2.1.31.1.1.1.1 # ifName
    overrides:
      ifAlias:
        ignore: true # Lookup metric
      ifDescr:
        ignore: true # Lookup metric
      ifName:
        ignore: true # Lookup metric
      ifType:
        type: EnumAsInfo
    version: 2
    auth:
      community: xxxx(snmp-agent设置的密码)

这里的oid信息就可以根据具体的厂商去查找了，例如华为： https://support.huawei.com/enterprise/zh/doc/EDOC1000178142/948603b1填写在配置文件的walk区域定义lookups和overrides给抓取的数据添加一些我们需要的labels定义auth和version信息，这个是是我们交换机前面已经设置好的这里只定义了我们需要walk的oid信息，但是实体信息是在mib里面的，比如

hwDev 来自 HUAWEI-MIB;
hwFrameIndex, hwSlotIndex 来自 HUAWEI-DEVICE-MIB;
OBJECT-GROUP, MODULE-COMPLIANCE 来自 SNMPv2-CONF;
Gauge, OBJECT-TYPE, MODULE-IDENTITY 来自 SNMPv2-SMI.
等等

那么mib在哪里有呢，这里提供了大量的mib库，https://github.com/librenms/librenms/blob/master/mibs在生成配置文件的时候，会尝试读取本地的mib库，$HOME/.snmp/mibs，因此我们把所需要的mib信息放在此处： $HOME/.snmp/mibs

验证页面http://xx.xx.xx.xx:9116输入generator配置里面的模块名称和想要抓取的目标网络设备的ip，即可查看抓取的所有指标

将指标接入prometheus

直接使用默认配置即可

scrape_configs:
  - job_name: 'snmp'
    static_configs:
      - targets:
        - 192.168.1.2  # SNMP device.
        - xx.xx.xx.xx
        - .....
    metrics_path: /snmp
    params:
      module: [if_mib]
    relabel_configs:
      - source_labels: [__address__]
        target_label: __param_target
      - source_labels: [__param_target]
        target_label: instance
      - target_label: __address__
        replacement: xx.xx.xx.xx:9116  # The SNMP exporter's real hostname:port.

重新加载prometheus的配置curl -X POST http://xx.xxx.xx.xx:9090/-/reload
