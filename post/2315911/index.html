基于drone的CI-CD实践

September 2, 2019

介绍

组件

  • drone：自助式的CI/CD交互平台，提供开源版本，可私有部署
  • helm：k8s的包管理器
  • k8s：云原生..
  • gitlab：代码仓库
  • maven/dockerhub：镜像仓库

流程

安装

安装方式推荐docker方式，一条命令完事儿，drone需要安装drone-server和drone-agent，参考：https://docs.drone.io/installation/gitlab/  

安装server：  

docker run \
  --volume=/var/run/docker.sock:/var/run/docker.sock \
  --volume=/var/lib/drone:/data \
  --env=DRONE_GIT_ALWAYS_AUTH=false \
  --env=DRONE_GITLAB_SERVER=http://gitlab.xxx.com \
  --env=DRONE_GITLAB_CLIENT_ID=9d71bdccd11156913a475891ab082f61677aae816e40911896cb82c17fcfc87e \
  --env=DRONE_GITLAB_CLIENT_SECRET=8039aff4d23560355e76e7855b409399c119fdb4005c9d02a2291bfe05d5b6da \
  --env=DRONE_RUNNER_CAPACITY=2 \
  --env=DRONE_SERVER_HOST=drone.test.com \
  --env=DRONE_USER_CREATE=username:xxx,admin:true\
  --env=DRONE_SERVER_PROTO=http \
  --env=DRONE_TLS_AUTOCERT=false \
  --env=DRONE_USER_FILTER=xxx \
  --env=DRONE_RPC_SECRET=b00553e8d490f13d296fa0f8a3bdd630 \
  --publish=80:80 \
  --publish=443:443 \
  --restart=always \
  --detach=true \
  --name=drone \
  drone/drone:1.2.3-linux-amd64

安装agent：  

  docker run -d \
  -e DRONE_RPC_PROTO=http \
  -e DRONE_RPC_HOST=drone.test.com \
  -e DRONE_RPC_SECRET=b00553e8d490f13d296fa0f8a3bdd630 \
  -e DRONE_RUNNER_CAPACITY=2 \
  -e DRONE_RUNNER_NAME=${HOSTNAME} \
  -p 3001:3000 \
  -v /var/run/docker.sock:/var/run/docker.sock \
  --restart always \
  --name runner \
  drone/agent:1.2.3-linux-amd64

使用

自动触发

首先，登陆安装时候在server那里指定的变量（DRONE_GITLAB_SERVER）对应的地址：http://drone.test.com ,会自动跳转到gitlab进行认证授权  然后会跳转到drone的界面，这里我们可以激活需要做cicd的项目  要做cicd的开发，只需要在项目的根目录添加 ** .drone.yml ** 文件（也支持自定义的文件）

kind: pipeline
name: test
clone:
  depth: 50
trigger:
#  branch:
#  - master
  event:
    include:
    - push
    - tag
    - promote
steps:
- name: go-build
  image: golang
  commands:
  - go build -o hello
  - ls -l
  - ./hello
 
- name: docker  
  image: plugins/docker
  settings:
    username: xxx
    password: xxx
    repo: imdingtalk/hello
    tags: ${DRONE_TAG}
  #其他内置变量参考
  #https://docs.drone.io/reference/environ/
# 这里发布的时候，使用自己打包的helm+kubectl的镜像，包含密钥信息
# 测试环境直接发布，不需要手动触发，生产环境一般手动触发
- name: helm-kubectl-deploy
  image: xxx/helm:3.0
  commands:
  - helm ls
  - kubectl get pod -n ops
  - helm  upgrade  --set image.tag=${DRONE_TAG}  hello stable/hello
  - helm history hello
  
##以下是测试手动触发任务，在自动触发时候不会执行以下任务，因为有when条件
- name: production
  image: golang
  commands:
  - echo "production"
  when:
    event: [ promote ]
    target: [ prod ]

- name: testing
  image: golang
  commands:
  - echo "testing"
  when:
    event: [ promote ]
    target: [ testing ]

手动触发

手动触发需要drone的客户端，一般用于生产部署    

drone build promote  xxx/drone  130    prod
这里的130，和prod对应前面.drone.yml文件中的when条件，触发对应的任务
#USAGE:
#   drone build promote [command options] <repo/name> <build> <environment>
